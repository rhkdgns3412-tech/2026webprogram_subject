# Requirements_Analysis



### 1. 회원가입 및 로그인
- JSP를 통해 사용자 입력을 받는다.
- Servlet에서 요청을 처리하고 DAO를 통해 DB에 저장한다.
- 로그인 성공 시 Session에 사용자 정보를 저장한다.

---

### 2. 후기 작성 기능
- 사용자는 거래 완료된 상품에 대해 후기 작성 가능
- ReviewServlet에서 요청을 받아 DB에 저장
- 한 상품에 대해 중복 후기 작성 방지 로직 적용

---

### 3. 평점 계산 기능
- 특정 사용자에 대한 모든 리뷰를 조회
- 평균 평점 계산 후 화면에 출력

예시 SQL:
SELECT AVG(rating) FROM reviews WHERE target_user_id = ?

---

### 4. 후기 조회 기능
- 특정 사용자의 후기 리스트 조회
- JSP에서 리스트 형태로 출력

## 상품관리 기능적 요구사항
- |요구사항 | 우선순위
- |상품등록 | 높음 | 
- | 상품삭제 | 높음|
- | 상품수정| 높음|
- | 상품조회 | 높음|
- | 상품찾기 | 높음 |

## 비기능적 요구사항
상품 등록시 대분류,중분류,소분류에따른 상품 분류
사용자가 정한 우선순위 대로 상품 나열
